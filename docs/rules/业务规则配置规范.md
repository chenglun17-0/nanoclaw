# 业务规则配置规范

## 1. 目的与适用范围

本规范定义 NanoClaw 的业务配置项、字段约束、默认值、校验规则与权限边界。  
目标是让“配置可查、行为可测、问题可回滚”。

## 2. 配置来源与优先级

| 级别 | 来源 | 说明 |
|---|---|---|
| P0 | 运行时 `process.env` | 最高优先级；`src/config.ts` 中多数项直接读取此来源 |
| P1 | 项目根目录 `.env` | 仅在调用 `readEnvFile(keys)` 的白名单 key 中生效 |
| P2 | 代码默认值 | `src/config.ts` 内默认常量 |

说明：当前在 `src/config.ts` 中，仅 `ASSISTANT_NAME`、`ASSISTANT_HAS_OWN_NUMBER` 通过 `.env` 白名单读取；其他常量默认直接走 `process.env` 或代码默认值。通道侧密钥（如飞书）由对应 channel 文件按需通过 `readEnvFile` 读取。`readEnvFile` 禁止全量装载环境变量。

## 3. 全局配置项规范（`src/config.ts`）

| 配置项 | 类型 | 默认值 | 约束/说明 | 主要影响模块 |
|---|---|---|---|---|
| `ASSISTANT_NAME` | string | `Andy` | 用于生成触发正则 `^@name\\b`（忽略大小写） | `src/index.ts`, `src/channels/*` |
| `ASSISTANT_HAS_OWN_NUMBER` | boolean | `false` | 字符串 `'true'` 才视为 true | 通道逻辑 |
| `POLL_INTERVAL` | number(ms) | `2000` | 消息轮询周期 | 主消息循环 |
| `SCHEDULER_POLL_INTERVAL` | number(ms) | `60000` | 调度轮询周期 | 任务调度器 |
| `CONTAINER_IMAGE` | string | `nanoclaw-agent:latest` | 容器镜像名 | 容器执行 |
| `CONTAINER_TIMEOUT` | number(ms) | `1800000` | 容器硬超时候选值 | 容器执行 |
| `IDLE_TIMEOUT` | number(ms) | `1800000` | 流式输出空闲关闭时长 | 容器执行、消息处理 |
| `MAX_CONCURRENT_CONTAINERS` | number | `5` | 最小值 1（`Math.max(1, ...)`） | 队列并发 |
| `TIMEZONE` | string | 系统时区 | cron 解析时区 | IPC/调度 |
| `MOUNT_ALLOWLIST_PATH` | string | `~/.config/nanoclaw/mount-allowlist.json` | 位于项目外部，不挂载进容器 | 挂载安全 |

### 3.1 飞书通道配置（`src/channels/feishu.ts`）

| 配置项 | 类型 | 必填 | 默认值 | 规则 |
|---|---|---|---|---|
| `FEISHU_APP_ID` | string | 是 | 无 | 通过 `readEnvFile` 从 `.env` 读取；缺失时通道启动失败 |
| `FEISHU_APP_SECRET` | string | 是 | 无 | 通过 `readEnvFile` 从 `.env` 读取；缺失时通道启动失败 |

## 4. 群组注册配置规范（`registered_groups`）

### 4.1 字段定义

| 字段 | 类型 | 必填 | 默认值 | 规则 |
|---|---|---|---|---|
| `jid` | string | 是 | 无 | 主键 |
| `name` | string | 是 | 无 | 展示名 |
| `folder` | string | 是 | 无 | 必须满足 `^[A-Za-z0-9][A-Za-z0-9_-]{0,63}$`，禁止 `/`、`\\`、`..`，禁止保留名 `global` |
| `trigger` | string | 是 | 无 | 当前用于持久化兼容；触发判定以全局 `TRIGGER_PATTERN` 为准 |
| `added_at` | ISO timestamp | 是 | 无 | 注册时间 |
| `requiresTrigger` | boolean | 否 | `true` | 非主组建议保持 true |
| `containerConfig` | object | 否 | 无 | 见 4.2 |

### 4.2 `containerConfig` 子项

| 字段 | 类型 | 默认值 | 规则 |
|---|---|---|---|
| `timeout` | number(ms) | 继承 `CONTAINER_TIMEOUT` | 实际硬超时为 `max(timeout, IDLE_TIMEOUT + 30000)` |
| `additionalMounts[]` | array | 空 | 每项必须通过 allowlist 校验，未通过即拒绝挂载并记录警告 |
| `additionalMounts[].hostPath` | string | 无 | 支持 `~` 展开；必须存在且在允许根目录下 |
| `additionalMounts[].containerPath` | string | `basename(hostPath)` | 必须是相对路径，非空，且不含 `..` |
| `additionalMounts[].readonly` | boolean | `true` | 若请求写入但策略不允许，将被强制只读 |

### 4.3 额外挂载安全规则

1. allowlist 文件缺失/解析失败时，`additionalMounts` 全部拒绝。
2. 必须同时通过：
   - 不命中 blocked patterns；
   - 位于 `allowedRoots` 某根目录内；
   - 容器路径合法。
3. `nonMainReadOnly=true` 时，非主组写挂载请求一律降级只读。

## 5. 调度任务配置规范（`scheduled_tasks`）

### 5.1 字段定义

| 字段 | 类型 | 必填 | 默认值 | 规则 |
|---|---|---|---|---|
| `id` | string | 是 | `task-{timestamp}-{rand}` | 主键 |
| `group_folder` | string | 是 | 无 | 必须对应已注册组 |
| `chat_jid` | string | 是 | 无 | 任务结果回传目标 |
| `prompt` | string | 是 | 无 | 执行内容 |
| `schedule_type` | enum | 是 | 无 | 业务约定值：`cron` / `interval` / `once`（当前 IPC 层未对未知值做硬拒绝） |
| `schedule_value` | string | 是 | 无 | 依 schedule_type 解释 |
| `context_mode` | enum | 否 | `isolated` | 仅 `group` / `isolated` |
| `status` | enum | 否 | `active` | `active` / `paused` / `completed` |
| `next_run` | ISO timestamp/null | 否 | 由 schedule 计算 | 到期判断依据 |

### 5.2 调度值校验规则

| `schedule_type` | `schedule_value` 规则 | 非法处理 |
|---|---|---|
| `cron` | 必须可被 `cron-parser` 按 `TIMEZONE` 解析 | 解析失败会拒绝并记录警告 |
| `interval` | 当前用 `parseInt` 解析，调用方应传纯数字且 `> 0` 的毫秒字符串 | 非正数会拒绝并记录警告 |
| `once` | 必须是可解析时间戳 | 解析失败会拒绝并记录警告 |
| 其他值 | 当前实现不会在 IPC 层直接报错 | 会写入任务但通常无法形成有效调度，应视为非法输入 |

## 6. IPC 指令与权限规范（`src/ipc.ts`）

| 指令 | 非主组权限 | 主组权限 | 备注 |
|---|---|---|---|
| `message` | 仅可发往自己组对应 `chatJid` | 当前可发往任意 `chatJid` | 来源目录即身份 |
| `schedule_task` | 仅可为自己组创建任务 | 可为任意已注册组创建任务 | 目标组必须已注册 |
| `pause_task` / `resume_task` / `cancel_task` | 仅可操作自己组任务 | 可操作所有任务 | 需先按 taskId 查归属 |
| `refresh_groups` | 禁止 | 允许 | 触发群元数据刷新（当前飞书实现为 no-op，仅重写快照） |
| `register_group` | 禁止 | 允许 | `folder` 必须过合法性校验 |

## 7. 推荐配置示例

```json
{
  "name": "研发群",
  "folder": "dev-team",
  "trigger": "@Andy",
  "requiresTrigger": true,
  "containerConfig": {
    "timeout": 600000,
    "additionalMounts": [
      {
        "hostPath": "~/projects/webapp",
        "containerPath": "webapp",
        "readonly": false
      }
    ]
  }
}
```

## 8. 变更与回滚规范

1. 新增配置字段时必须同步：
   - 更新 `src/types.ts` 类型定义；
   - 更新 `src/ipc.ts` / `src/db.ts` 读写链路；
   - 更新本规范与 `docs/rules/project.md`。
2. 涉及挂载策略、权限策略、超时策略变更时，必须提供回滚方案：
   - 回滚触发条件；
   - 回滚步骤；
   - 回滚后验证项（任务执行、消息发送、挂载生效）。
3. 任何“放宽权限”的改动必须附带测试用例，至少覆盖未授权场景。

## 9. ⚠️ 严禁事项

- 严禁绕过 IPC 授权矩阵直接操作跨组任务。
- 严禁把机密目录加入 allowlist 且允许读写。
- 严禁接受未校验的 `folder` / `containerPath` 入库或用于路径拼接。
