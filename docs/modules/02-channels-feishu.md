# 通道接入（飞书）模块

[返回模块索引](../MODULES.md)

## 模块定位

提供飞书渠道的长连接接入、消息收发能力，并向主编排暴露统一通道行为。

## 关键文件

- `src/channels/feishu.ts`

## 主要职责

1. 维护飞书连接生命周期（连接、自动重连、断开）。
2. 解析入站事件并回调主流程。
3. 管理出站发送、排队与重试。
4. 通过事件上报会话元数据（群/单聊、最后活跃时间）。

## 输入与输出

- 输入：飞书事件订阅消息、主编排发起的发送请求。
- 输出：结构化入站消息、通道状态更新、出站发送结果。

## 改动建议

1. 通道协议逻辑与业务策略分离，避免在此模块新增任务/权限逻辑。
2. 事件处理必须保持快速返回，避免阻塞导致飞书重推。
