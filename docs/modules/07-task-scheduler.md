# 任务调度模块

[返回模块索引](../MODULES.md)

## 模块定位

负责定时任务生命周期：发现到期任务、执行任务容器、记录结果与下一次运行时间。

## 关键文件

- `src/task-scheduler.ts`

## 主要职责

1. 周期轮询到期任务（cron/interval/once）。
2. 通过容器执行链路运行任务。
3. 将结果回传到对应聊天并记录任务运行日志。
4. 更新任务状态、`next_run` 和摘要结果。

## 输入与输出

- 输入：DB 中 active 任务、当前时间、group 上下文。
- 输出：任务执行记录、聊天消息、任务状态更新。

## 改动建议

1. 调整调度间隔前先评估 DB 轮询压力与消息延迟。
2. 修改任务结果回传行为时，注意避免与主消息流重复发送。
