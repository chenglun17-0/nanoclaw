# 路由与格式化模块

[返回模块索引](../MODULES.md)

## 模块定位

负责消息内容在“通道输入格式”与“Agent 输入格式”之间的转换，并统一出站清理逻辑。

## 关键文件

- `src/router.ts`

## 主要职责

1. 将消息列表格式化为结构化输入（当前为 XML 包裹格式）。
2. 清理 Agent 输出中的内部标签（如 `<internal>`）。
3. 根据 JID 选择对应可用通道发送消息。

## 输入与输出

- 输入：`NewMessage[]`、原始 Agent 输出文本、目标 JID。
- 输出：格式化 prompt 文本、清理后的出站文本、路由后的发送调用。

## 改动建议

1. 保持纯函数优先，减少状态依赖，便于测试。
2. 修改输出清理规则时，联动检查消息重复发送和空消息过滤逻辑。
