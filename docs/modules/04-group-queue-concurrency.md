# 组队列与并发控制模块

[返回模块索引](../MODULES.md)

## 模块定位

统一管理消息处理与任务执行的并发策略，保证 group 内串行、全局并发可控。

## 关键文件

- `src/group-queue.ts`

## 主要职责

1. 维护 group 级执行状态与全局活动容器计数。
2. 对消息触发与任务触发统一排队。
3. 在容器 idle 时触发关闭，避免资源长期占用。
4. 处理失败重试与阻塞队列排空。

## 输入与输出

- 输入：主编排的消息检查请求、调度器的任务执行请求。
- 输出：触发容器执行、关闭信号写入、排队状态更新。

## 改动建议

1. 变更并发参数时，先评估对任务调度和消息延迟的连锁影响。
2. 新增队列状态字段时，注意与 `closeStdin`/`notifyIdle` 的时序一致性。
